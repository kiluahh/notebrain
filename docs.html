Obsidian Plugin

Notebrain

AI Agent inside your Obsidian notes.

Chat with DeepSeek, search the web, manage your vault, research papers, and more — all from within any note. No context switching. No separate chat window. Your notes are the interface.

Get Started Browse 21 Tools

Getting Started

  1. Install the plugin
    Open Obsidian Settings → Community Plugins → Browse, search for Notebrain, and install. Alternatively, download the latest release from GitHub and place main.js, manifest.json, and styles.css in your vault's .obsidian/plugins/notebrain/ folder.
  2. Get a DeepSeek API key
    Visit platform.deepseek.com, sign up, and create an API key. Paste it into the plugin settings under the Model tab.
  3. Open any note and start chatting
    Press Ctrl + Shift + Enter (Mac: Cmd + Shift + Enter) to open the inline chat input.
  4. Type your message and press Enter
    Your message is sent to DeepSeek. The AI streams its response directly into your note. Thinking content appears inside %% comment blocks.

Inline Chat

Notebrain's chat lives inside your notes using Obsidian's %% comment markers for thinking content and tool calls. There is no separate chat panel — your note is both the input and output surface.

How It Works

  1. Place your cursor in any markdown note and press Ctrl + Shift + Enter.
  2. An input prompt appears at the bottom of the note. Type your message.
  3. The plugin writes a header to the file, then streams the AI response directly to the file in 80ms batches (configurable).
  4. Thinking/reasoning content appears inside %%...%% comment blocks — invisible in reading mode, visible in source/live preview.
  5. Tool calls are displayed in the thinking block. Tool results are passed to the AI but not written to the note (keeps the note clean).
  6. When the AI produces a real response, the %% block closes and the message appears as normal text.

Chat Format

Note Content 
%%
user
[Timestamp: 2026-05-19 14:30:00 +08:00]
%%

What are the key ideas in quantum computing?

%%
[Thinking]
Let me break down quantum computing into its
fundamental concepts: superposition, entanglement,
and interference...
[Tool call: searchNotes {query:"quantum computing"}]
%%

Quantum computing leverages three core principles:

1. Superposition: Qubits exist in multiple states...
2. Entanglement: Correlated qubits share state...
3. Interference: Wave-like probability manipulation...

%%
assistant
[Timestamp: 2026-05-19 14:32:15 +08:00]
%%

Key Behaviors

  • Unified file-only streaming: All content is written through vault.modify(). The file is the single source of truth, eliminating race conditions.
  • 30-minute chat cache: Full API-native messages (with tool calls and reasoning) are cached in memory. Within the 30-minute window, subsequent messages reuse the cache for efficiency.
  • Tool permission modal: Destructive operations like createNote and editNote show a confirmation dialog before execution.
  • Session tags: After each chat session, #notebrain-session is automatically added to the note's frontmatter tags.
Tip

You can also trigger inline chat by clicking the brain icon in the left ribbon, or via the command palette (Notebrain: Chat).

FIM Completion

Fill-in-the-Middle (FIM) lets you insert a +++ marker anywhere in your note and have the AI generate text to fill the gap between the content above (prefix) and below (suffix).

Usage

  1. Place +++ on its own line at the position where you want AI-generated content.
  2. Place your cursor on that line.
  3. Trigger FIM from the ribbon (pen icon) or command palette (Notebrain: FIM Complete).
  4. The AI fills the gap and replaces the +++ marker with the generated text.

Custom Token Count

Append a number to the marker to set a custom max token count for that specific completion:

Prefix/content above the marker
Chapter 1: The Beginning

The old library stood silent...

+++512

Chapter 3 would reveal the truth.

Here, +++512 requests up to 512 tokens to bridge the gap. The default is 1024 tokens (configurable in FIM settings). The FIM API has its own dedicated settings — separate API key, base URL, model, temperature, and max tokens — independent of the chat model configuration.

How FIM works

The text above +++ becomes the prefix and the text below becomes the suffix. The model generates content that fits naturally between them. If there is no text after +++, the model does suffix-only completion (continuation).

NOTEBRAIN.md

NOTEBRAIN.md is a vault-wide AI instructions file, similar to CLAUDE.md or AGENTS.md. It is automatically created in your vault root on first load if it does not exist. The path is configurable in settings.

Use NOTEBRAIN.md to set rules, preferences, and context that the AI follows across all your notes. The content is injected into the system prompt at the start of every chat session.

Example NOTEBRAIN.md

NOTEBRAIN.md
# NOTEBRAIN.md

## AI Behavior Rules
- Language: Always respond in English unless the user writes in Chinese.
- Style: Be concise. Prefer bullet points over long paragraphs.
- Tone: Professional but friendly. Avoid marketing fluff.

## Vault Structure
- /Projects/ — active project notes (do not create here unless asked)
- /Archive/ — completed/archived content
- /Daily/ — daily journal entries (template: YYYY-MM-DD.md)
- /References/ — reference material, wiki-style notes

## Preferences
- When creating notes, use the Daily template.
- Do not create notes in the root folder.
- Use wikilinks for cross-references.

You can also store NOTEBRAIN.md in a subfolder (e.g., Config/NOTEBRAIN.md) — just update the path in settings. The plugin auto-discovers any file named "notebrain" (case-insensitive) in your vault and offers them in a dropdown.

Settings Guide

Notebrain settings are organized into five tabs. Open them via Settings → Community Plugins → Notebrain → gear icon.

Basic

General plugin configuration — language, instruction file path, and streaming behavior.

Language
Settings UI display language.
Options: Chinese (zh) or English (en). Affects settings labels, tool names, and descriptions in the settings UI. Does not affect AI output language — control that via NOTEBRAIN.md.
Default: zh
NOTEBRAIN.md Path
Where the AI instructions file is stored.
Dropdown lists all markdown files named "notebrain" (case-insensitive) in your vault. The root NOTEBRAIN.md is always available as the first option. Contents are injected into the system prompt.
Default: NOTEBRAIN.md (root)
Write Frequency
Milliseconds between file writes during streaming.
Controls how often the AI response is flushed to disk during streaming. Lower values = smoother display but higher CPU usage. Must be between 20–500ms.
Default: 80ms

Model

API connection and model behavior settings for the chat agent.

API Key
DeepSeek API key for authentication.
Get your key from platform.deepseek.com. Stored locally in your Obsidian data folder — never sent anywhere except to the configured base URL.
Required
Base URL
API endpoint address.
Default points to DeepSeek's official API. Change this to use OpenAI-compatible endpoints (e.g., local LLM servers, proxies, or other providers).
Default: https://api.deepseek.com
Model
Model identifier string.
Text input with available-models discovery. The plugin attempts to fetch the model list from the API on settings load and shows available models inline. Supports any DeepSeek model and OpenAI-compatible models.
Default: deepseek-v4-pro
Thinking Mode
Enable reasoning/thinking output.
When enabled, the model outputs its reasoning process inside %%...%% comment blocks. Thinking content is hidden in reading mode but visible in source/live preview. Crucial: reasoning content must be passed back in subsequent messages after tool calls, or the API returns a 400 error — the cache handles this automatically for warm starts.
Default: On
Reasoning Effort
Thinking intensity level.
Dropdown: high or max. Higher effort produces more thorough reasoning but takes longer and uses more tokens. Only applies when Thinking Mode is on.
Default: high
Temperature
Creativity level (0–2).
Controls randomness in output. Lower = more focused/deterministic. Higher = more creative/varied. Only applies when Thinking Mode is off.
Default: 0.7

Tools

API keys for external services and per-tool enable/disable toggles.

External API Keys

Tavily API Key
Required for Tavily web search, extract, crawl, map, and research tools. Get one at tavily.com. Format: tvly-...
GitHub Token
Optional personal access token for higher API rate limits when using githubSearch. Create at github.com/settings/tokens. Format: ghp_...
Baidu AppBuilder Key
Required for baiduBaike (Baidu Encyclopedia) lookups. Get one at appbuilder.baidu.com. Format: bce-...
Headless Browser Endpoint
Optional custom headless browser endpoint for webFetch with renderJs=true. If left empty, the built-in free renderer (Jina AI) is used.

Tool Toggles (21 Tools)

Each tool has an individual enable/disable toggle. Disabled tools are hidden from the AI — it cannot see or call them. Toggle off tools you do not use to reduce token usage in the system prompt.

Note

Controls for note creation, editing permissions, and templates.

Enable YAML
Allow AI to create notes with YAML frontmatter.
When off, the AI is forbidden from creating notes with YAML frontmatter. Any frontmatter in generated content is automatically stripped before writing.
Default: On
Edit Permission Tag
Tag required for AI to edit a note.
Only notes containing this tag (in body or frontmatter) can be edited by the AI via createNote and editNote. Auto-prepends # if you forget it. Change this to restrict AI editing to specific notes only.
Default: #notebrain
Note Template
Template note for new note creation.
Dropdown lists all markdown files in your vault. When set, newly created notes (via createNote) will be based on the selected template's structure. Leave empty for no template.
Default: (none)

FIM

Separate API configuration for Fill-in-the-Middle completion. FIM settings are completely independent from the chat model settings — you can use a different API key, endpoint, or model.

API Settings

API Key
FIM-specific API key. Leave empty to use the Model tab's API key.
Base URL
FIM API endpoint. /beta is automatically appended. Uses a separate base URL from the chat model.
Default: https://api.deepseek.com
Model
Model for FIM completions. Leave empty to use the Model tab's model.
Default: deepseek-v4-pro

Parameters

Temperature
Creativity level for FIM completions (0–2). Lower values produce more focused, deterministic output.
Default: 1.0
Max Tokens
Default maximum completion length (1–4096). Can be overridden per-marker with +++N syntax.
Default: 1024

Tools Reference

Notebrain ships with 21 tools across six categories. The AI decides which tools to call based on your message. You can call multiple tools in parallel when they are independent.

Vault 6 tools

searchNotes
Search the entire vault — content, tags, filename, path, properties, and regex. Returns matched notes with snippets.
readNote
Read a note by name or path. Returns full content, metadata, wikilinks, and heading structure.
listFolder
List folder contents (files, subfolders). Useful for browsing vault structure and discovering notes.
getBacklinks
Get all notes that link to a given note. Essential for understanding relationships in your knowledge graph.
createNote
Create a new note at a specified path. Requires user confirmation via modal. Respects note template setting.
Requires: edit permission tag
editNote
Edit an existing note — replace, append, or prepend content. Requires user confirmation via modal.
Requires: edit permission tag

Time 1 tool

getCurrentTime
Get the current date and time with timezone support. Returns ISO 8601 formatted timestamp.

Web Free 3 tools

webSearch
Free web search using Baidu + Bing. Requires at least 3 keywords, searches both engines per keyword. No API key needed.
webFetch
Fetch and extract readable content from any URL. Supports renderJs=true for JavaScript-heavy pages via built-in free renderer (Jina AI) or custom headless browser endpoint.
cnNews
Aggregate headlines from 23 major Chinese news sites (Xinhua, People's Daily, CCTV, etc.). Optional tier parameter for filtering by importance.

Tavily 6 tools Requires: Tavily API Key

tavilySearch
Search the web with configurable depth (basic/advanced), topic filtering, and domain include/exclude lists.
tavilyExtract
Extract clean, structured content from one or more URLs. Returns markdown or text with advanced extraction for tables.
tavilyCrawl
Crawl a website starting from a URL with configurable depth and breadth. Follows links and extracts content.
tavilyMap
Map a website's structure. Returns a list of all URLs discovered starting from a base URL. Good for site exploration.
tavilyResearch
Multi-step deep research on any topic. Polls for up to 3 minutes. Returns a comprehensive research report.
tavilyResearchGet
Retrieve a completed Tavily deep research report by its request_id.

Academic 4 tools

githubSearch
Search GitHub repositories, code, issues, and users. Works without a token. Optional token for higher rate limits.

Search arXiv preprints with full search_query syntax (e.g., cat:cs.LG AND ti:diffusion). Returns titles, authors, abstracts. Shared 3-second rate limit with arxivDownload.
arxivDownload
Download paper as HTML preview (for reading) or PDF. Files saved to vault root. Shared 3-second rate limit with arxivSearch.
wikipedia
Search Wikipedia or get full article content. Supports multiple languages (lang parameter). No API key required.

Query Baidu Encyclopedia (Baidu Baike) entries by title or ID. Returns structured encyclopedia content in Chinese.

FAQ

Is Notebrain free?
The plugin is free and open source (MIT). However, the AI features require a DeepSeek API key, which is a paid service. DeepSeek's pricing is competitive — typically a fraction of the cost of other major AI providers. Some tools (webSearch, webFetch, cnNews, githubSearch, arxivSearch, arxivDownload, wikipedia) work without additional API keys.
Does it work on mobile?
Yes. Notebrain is fully mobile-compatible (Obsidian Android and iOS). It uses vanilla DOM only (no React), requestUrl instead of fetch (required by Obsidian's CSP on mobile), and pauses file writes during tool execution to avoid single-threaded I/O deadlocks on mobile devices. The isDesktopOnly flag in the manifest is set to false.
What models are supported?
Any DeepSeek model — deepseek-v4-pro, deepseek-v4-flash, and future models. Because the plugin uses the OpenAI-compatible chat completions API, you can point the Base URL to any OpenAI-compatible endpoint — including local models (Ollama, LM Studio), proxies, or other providers. The model list is auto-discovered from your configured endpoint.
Can the AI edit my notes?
Yes, with permission. The createNote and editNote tools can modify your vault, but they require two levels of protection: (1) the target note must contain the edit permission tag (#notebrain by default, configurable), and (2) a confirmation modal appears before any destructive operation. You can disable these tools entirely from the Tools tab if you prefer.
Is my data private?
Your API key is stored locally in Obsidian's data folder and is never sent anywhere except to your configured Base URL. The content of your messages is sent to the DeepSeek API (or your configured endpoint) for processing. Review DeepSeek's privacy policy for details on how they handle API data. Note content is only read when explicitly requested (search, read, edit tools) and only within your vault.
What is the chat cache and how does it work?
Notebrain maintains a 30-minute in-memory cache per file. Full API-native messages (with tool calls, tool results, and reasoning content) are stored so that when you send a follow-up message within 30 minutes, the plugin can reconstruct the full conversation context without re-parsing the entire note. If the cache expires, the plugin rebuilds context from the note's user and assistant messages (without thinking history, which is not needed for new turns). The cache resets on each new message within the window.
Why does the plugin use %% comments for thinking content?
Obsidian's %%...%% syntax creates comments — content that is visible in source/live preview mode but hidden in reading mode. This keeps your notes clean and readable while still preserving the AI's reasoning trace for debugging or review. The thinking content is also required by the DeepSeek API: when thinking mode is on, reasoning_content must be passed back in subsequent messages after tool calls, or the API returns a 400 error.
