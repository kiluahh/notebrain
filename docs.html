Getting Started

Notebrain turns any Obsidian note into an AI-powered workspace. Follow these steps to set it up in under two minutes.

  1. Install the plugin
    Open Obsidian Settings → Community Plugins → Browse, search for Notebrain, and install. Alternatively, download the latest release from GitHub and place main.js, manifest.json, and styles.css in your vault's .obsidian/plugins/notebrain/ folder.
  2. Get a DeepSeek API key
    Visit platform.deepseek.com, sign up, and create an API key. Paste it into the plugin settings under the Model tab. The key is stored locally on your device and never sent anywhere except your configured API endpoint.
  3. Review your NOTEBRAIN.md (auto-created)
    On first load, the plugin automatically creates a NOTEBRAIN.md file in your vault root. This file contains AI behavior rules and preferences that apply to all chat sessions. You can customize it freely — set your preferred language, writing style, vault structure, and any rules the AI should follow. The file path is configurable in settings.
  4. Choose your language
    The settings UI supports both Chinese (zh) and English (en). This setting affects the settings panel labels only — not the AI output language. To control the AI's response language, add a rule in your NOTEBRAIN.md (e.g., "Always respond in English" or "Reply in the same language as the user").
  5. Open any note and start chatting
    Place your cursor in any markdown note and press Ctrl + Shift + Enter (Mac: Cmd + Shift + Enter). Type your message and press Enter. The AI streams its response directly into your note. You can also trigger chat from the left ribbon icon or the command palette (Notebrain: Chat).

Inline Chat

Notebrain's chat lives inside your notes using Obsidian's %% comment markers for thinking content and tool calls. There is no separate chat panel — your note is both the input and output surface.

Why Comments?

Obsidian's %%...%% syntax creates comments — content that is visible in source/live preview mode but completely hidden in reading mode. This means:

  • AI thinking traces and tool calls are tucked away inside comments — your note stays clean for reading and sharing.
  • You can still inspect the reasoning in source mode if you want to debug or review the AI's thought process.
  • No special UI elements in the note — just standard Obsidian markdown.

Chat Format

Each chat turn consists of a user message, an optional thinking/tool-call block, and an AI response. The thinking block uses a single %%...%% pair that stays open during the agent loop and closes when a real response appears.

Note Content 
%%
user
[Timestamp: 2026-05-19 14:30:00 +08:00]
%%

What are the key ideas in quantum computing?

%%
[Thinking]
Let me break this down: quantum computing rests on
superposition, entanglement, and interference...
[Tool call: searchNotes {"query":"quantum computing"}]
%%

Quantum computing leverages three core principles:

1. Superposition: Qubits exist in multiple states
   simultaneously, enabling exponential parallelism.
2. Entanglement: Correlated qubits share state
   regardless of distance — essential for error correction.
3. Interference: Wave-like probability amplitudes
   are manipulated to amplify correct answers.

%%
assistant
[Timestamp: 2026-05-19 14:32:15 +08:00]
%%

The Agent Loop

Notebrain implements a full agent loop: the AI can call tools, receive results, and decide on further actions — all within a single user message. Key behaviors:

  • Up to 10 tool-call turns per message. The AI can chain multiple tool invocations (e.g., search notes, then read a specific note, then search the web) within one agent turn.
  • Parallel tool calls are supported. If the AI calls two independent tools (e.g., searchNotes and webSearch), they execute concurrently.
  • Tool results are NOT written to the note. They are passed to the AI in private API messages, keeping your notes clean. Only the final AI response text appears in the note.
  • Unified file-only streaming: all content is written through vault.modify(). The file is the single source of truth, which eliminates race conditions with auto-merge and editor-staleness on tab switch. Tool execution pauses file writes to avoid I/O deadlocks on mobile.

Tool Permission Modal

Destructive operations — specifically createNote and editNote — require explicit user approval before execution. When the AI wants to create or modify a note, a modal dialog appears showing the exact operation. You can approve or deny each action. Additionally, these tools only work on notes that contain the edit permission tag (default: #notebrain, configurable in settings).

Session Tags

After each chat session, the tag #notebrain-session is automatically added to the note's frontmatter. This makes it easy to find all notes where you've had AI conversations — just search for #notebrain-session in your vault.

Chat Cache (30-min TTL)

Notebrain maintains an in-memory cache per file. Full API-native messages (including tool calls, tool results, and reasoning content) are stored so that follow-up messages within 30 minutes can reuse the complete conversation context without re-parsing the note. Key points:

  • Cache hit: the plugin uses the cached messages directly, skipping the note parse step. This is faster and preserves full tool-call history.
  • Cache miss/expired: the plugin rebuilds context from the note's user and assistant messages only. Thinking history is not needed for new turns and is excluded.
  • Each new message within the 30-minute window resets the timer, extending the cache lifetime.
  • This is critical for thinking mode — the DeepSeek API requires reasoning_content to be passed back in subsequent messages after tool calls, or it returns a 400 error. The cache handles this automatically for warm starts.
Accessing Chat

You can trigger inline chat in three ways: the keyboard shortcut Ctrl+Shift+Enter, the brain icon in the left ribbon, or the command palette (Notebrain: Chat).

FIM Completion

Fill-in-the-Middle (FIM) lets you insert a +++ marker anywhere in your note and have the AI generate text to fill the gap between the content above (prefix) and below (suffix). It is ideal for code generation, writing continuation, or bridging two sections of a note.

Basic Usage

  1. Place +++ on its own line where you want AI-generated content.
  2. Place your cursor on that line.
  3. Trigger FIM from the ribbon (pen icon) or command palette (Notebrain: FIM Complete).
  4. The AI fills the gap and replaces the +++ marker.

How Prefix/Suffix Works

The text above +++ becomes the prefix — it tells the model what comes before the gap. The text below +++ becomes the suffix — it tells the model what should come after. The model generates content that flows naturally from the prefix into the suffix.

Custom Token Count

Append a number to the marker to set a custom max token count for that specific completion:

Custom Token Syntax 
Chapter 1: The Beginning

The old library stood silent, its ancient shelves
sagging under the weight of centuries...

+++512

Chapter 3 would reveal the truth Professor Hart
had spent a lifetime chasing.

Here, +++512 requests up to 512 tokens. The default is configured in FIM settings (default: 1024). The range is 1–4096.

NOTEBRAIN.md

NOTEBRAIN.md is a vault-wide AI instructions file, similar to CLAUDE.md or AGENTS.md in the coding world. It is automatically created in your vault root on first plugin load if it does not exist. The path is configurable in settings — you can move it to a subfolder if you prefer.

Use NOTEBRAIN.md to set rules, preferences, and context that the AI follows across all your notes. The content is injected into the system prompt at the start of every chat session. This means you can set a consistent personality, workflow, and constraints for the AI without repeating yourself in every message.

Template Example

NOTEBRAIN.md 
---
# Optional YAML frontmatter
language: en
style: concise
---

# NOTEBRAIN.md

## AI Behavior Rules
- Language: Always respond in English
  unless the user writes in Chinese.
- Style: Be concise and direct. Prefer bullet points
  over long paragraphs in responses.
- Tone: Professional but approachable. Avoid
  marketing fluff and buzzwords.
- Code: Always include language annotations in
  fenced code blocks. Add brief comments for complex logic.

## Vault Structure
- /Projects/ — active project notes
  (do not create notes here unless explicitly asked)
- /Archive/ — completed or archived content
- /Daily/ — daily journal entries
  (template: YYYY-MM-DD.md)
- /References/ — permanent knowledge,
  wiki-style notes
- /Templates/ — note templates
  (never modify without permission)

## Preferences
- When creating new notes, default to the /References/ folder.
- Do not create notes in the vault root.
- Use wikilinks ([[note name]]) for
  cross-references between notes.
- Prepend a YAML frontmatter block with tags
  and created date for new notes.
- When searching, prioritize notes by tags,
  then filename, then path.
- If uncertain, ask before modifying existing notes.

Search Priority

When the AI searches your vault (via searchNotes), it prioritizes results in this order:

  1. Tags — frontmatter tags and inline tags (#tag)
  2. Filename — note title matching
  3. Path — folder location matching
  4. Content — full-text body search
  5. Properties — YAML frontmatter fields
  6. Regex — advanced pattern matching

Discovery & Multiple Files

The plugin automatically discovers any file named "notebrain" (case-insensitive) in your vault and offers them all in a dropdown in the settings. This means you can have different instruction files for different purposes — though only one is active at a time, selected in the Basic settings tab.

Settings Guide

Notebrain settings are organized into five tabs. Open them via Settings → Community Plugins → Notebrain → gear icon. Settings are stored locally in your Obsidian data folder and never leave your device (except for API keys sent to their respective endpoints).

Basic Settings

General plugin configuration — language, instruction file path, and streaming behavior.

Language
Settings UI display language.
Toggles between Chinese (zh) and English (en). Affects settings labels, tool names, and descriptions in the settings UI only. Does NOT affect AI output language — control that via NOTEBRAIN.md.
Default: zh
NOTEBRAIN.md Path
Location of AI instruction file.
Dropdown listing all markdown files named "notebrain" (case-insensitive) in your vault. The root file is always the first option. Choose which instruction file to use. Contents are injected into the system prompt at the start of every chat session.
Default: NOTEBRAIN.md (vault root)
Write Frequency
Milliseconds between file writes during streaming.
Controls how often the AI response is flushed to disk during streaming. Lower values = smoother live display but higher CPU/disk usage. Must be between 20–500ms. On mobile, consider increasing this value to reduce I/O pressure.
Default: 80ms

Model Settings

API connection and model behavior. The API Key is required — all other settings have sensible defaults.

Critical

You must provide a valid API key for the plugin to function. Without it, chat and FIM will fail.

API Key
DeepSeek API key for authentication.
Get your key from platform.deepseek.com. Stored locally in your Obsidian data folder. Sent only to the configured Base URL. Never transmitted anywhere else.
Required
Base URL
API endpoint address.
Default points to DeepSeek's official API. Change this to use any OpenAI-compatible endpoint — local LLM servers (Ollama, LM Studio), proxies, or other providers.
Default: https://api.deepseek.com
Model
Model identifier string.
Text input with available-models discovery. The plugin attempts to fetch the model list from the API on settings load. Supports any DeepSeek model and OpenAI-compatible models.
Default: deepseek-v4-pro
Thinking Mode
Enable reasoning/thinking output.
When enabled, the model outputs its reasoning process inside %%...%% comment blocks — invisible in reading mode. Crucial: reasoning content must be passed back in subsequent messages after tool calls, or the API returns a 400 error. The chat cache handles this automatically for warm starts.
Default: On
Reasoning Effort
Thinking intensity level.
Dropdown: high or max. Higher effort produces more thorough reasoning but takes longer and uses more tokens. Only applies when Thinking Mode is on.
Default: high
Temperature
Creativity level (0–2).
Controls randomness in output. Lower = more focused/deterministic. Higher = more creative/varied. Note: temperature only applies when Thinking Mode is OFF. When thinking is enabled, the model's internal reasoning temperature is used instead.
Default: 0.7

Tools Reference

Most tools work without additional API keys. Only Tavily tools require a Tavily key, baiduBaike requires a Baidu AppBuilder key, and githubSearch benefits from an optional GitHub token for higher rate limits. All vault tools, time, free web tools, arXiv, and Wikipedia work out of the box.

FAQ

Is Notebrain free?
The plugin is free and open source (MIT license). However, the AI features require a DeepSeek API key, which is a paid service. DeepSeek's pricing is competitive — typically a fraction of other major AI providers. Many tools (webSearch, webFetch, cnNews, githubSearch, arxivSearch, arxivDownload, wikipedia) work without additional API keys.
Is my data sent to DeepSeek servers?
Yes. When you chat with the AI, the content of your messages (and any notes the AI reads via tools) is sent to the DeepSeek API (or your configured endpoint) for processing. This is how all AI chat plugins work — the model runs on remote servers. Review DeepSeek's privacy policy for details on how they handle API data. Your API key is stored locally and never shared with anyone except your configured endpoint.
Can I use other models besides DeepSeek?
Yes. Because the plugin uses the OpenAI-compatible chat completions API, you can point the Base URL to any OpenAI-compatible endpoint — including local models (Ollama, LM Studio), proxies, or other cloud providers. The model list is auto-discovered from your configured endpoint on settings load. You can also use different models for chat vs. FIM completion.
Does it work offline?
No. Notebrain requires an internet connection because it sends API calls to a remote AI model. If you point the Base URL to a local model server (e.g., Ollama running on your machine), you can work offline as long as the local server is running. Vault-only tools (searchNotes, readNote, listFolder, getBacklinks) could theoretically work offline, but the AI brain that calls them lives on the server side.
How do I get a DeepSeek API key?
Visit platform.deepseek.com, create an account, and navigate to the API Keys section. Generate a new key and paste it into the Notebrain settings under the Model tab. Top-up credits as needed — DeepSeek uses a pay-as-you-go model. Pricing details are available on their platform page.
What is the difference between Tavily and the free web search?
The free webSearch tool uses Baidu and Bing search engines without any API key — it is fast, free, and good for quick lookups. Tavily is a specialized search API designed for AI agents, offering higher-quality results, advanced extraction, website crawling, mapping, and multi-step deep research. Tavily requires an API key (free tier available). If you need research-quality web access, Tavily is recommended. For casual searches, the free tool is sufficient.
Can the AI accidentally delete or corrupt my notes?
No. The editNote and createNote tools have two layers of protection: (1) the target note must contain the edit permission tag (#notebrain by default, configurable), and (2) a confirmation modal appears before any destructive operation, showing exactly what will be changed. You must manually approve each action. The AI cannot bypass these safeguards. If you are still concerned, you can disable both tools entirely from the Tools settings tab.
Does it work on mobile (Android/iOS)?
Yes. Notebrain is fully mobile-compatible. It uses vanilla DOM only (no React or other heavy frameworks), requestUrl instead of fetch (required by Obsidian's CSP on mobile), and pauses file writes during tool execution to avoid single-threaded I/O deadlocks on mobile devices. The isDesktopOnly flag in the manifest is set to false.
What is the chat cache and why is it 30 minutes?
Notebrain maintains an in-memory cache per file that stores full API-native messages (including tool calls, tool results, and reasoning content). This allows follow-up messages to reuse the complete conversation context without re-parsing the note. The 30-minute TTL was chosen because: (1) it covers most natural conversation sessions, (2) it prevents stale cache from consuming memory indefinitely, and (3) after 30 minutes, the context is typically still recoverable from the note content. Each new message within the window resets the timer.
Why does the plugin use %% comments for thinking content?
Obsidian's %%...%% syntax creates comments — content visible in source/live preview mode but hidden in reading mode. This keeps your notes clean while preserving the AI's reasoning trace. Additionally, the DeepSeek API requires that reasoning_content be passed back in subsequent messages after tool calls (or the API returns a 400 error). The chat cache handles this automatically. If you switch to a different model that does not use thinking mode, you can disable it in settings.
How does the agent loop work? Can it get stuck in a loop?
The agent loop allows the AI to call tools, receive results, and decide on further actions within a single user message. It is capped at 10 tool-call turns per message to prevent infinite loops. The AI can call multiple tools in parallel when they are independent. Tool results are passed to the AI but not written to the note — only the final text response appears. If the AI exceeds 10 turns, it is forced to produce a final response.
Can I customize which tools the AI has access to?
Yes. Each of the 21 tools has an individual toggle in the Tools settings tab. Disabled tools are removed from the system prompt entirely — the AI cannot see or invoke them. This reduces token usage and gives you fine-grained control over what the AI can do. For example, you can disable all web tools if you only want vault-only interactions, or disable createNote/editNote if you want read-only AI behavior.

Tips & Tricks

Get the most out of Notebrain with these practical tips from power users.

📝
Customize AI Behavior with NOTEBRAIN.md
Set language preferences, writing style rules, vault structure conventions, and behavioral constraints in NOTEBRAIN.md. The AI reads this at the start of every session. A well-written NOTEBRAIN.md is the single biggest lever for getting consistent, high-quality results.
⚡
Use +++ for Quick Code Completion
The FIM marker (+++) is the fastest way to get AI-generated code or text inserted inline. Place it between prefix and suffix content, trigger with the ribbon icon or command palette, and the AI fills the gap instantly. Append a number (+++256) to control token count.
🧠
Enable Thinking Mode for Complex Tasks
For reasoning-heavy tasks (math, logic, coding, multi-step research), keep Thinking Mode on. The AI's internal reasoning improves accuracy significantly. For simple Q&A or quick lookups, turning it off can speed up responses and reduce token usage.
🔍
Use Tavily for Research-Quality Web Search
The free web search is good for quick lookups, but Tavily provides higher-quality results with better source attribution. If you do frequent web research, get a Tavily API key (free tier available) for significantly better search and extraction quality.
🎛
Toggle Off Unused Tools
Every enabled tool adds to the system prompt size, consuming tokens and context window space. In the Tools settings tab, disable any tools you do not use (e.g., disable academic tools if you never search papers). This saves cost and leaves more context for the conversation.
📊
Adjust Write Frequency for Performance
On desktop, lower write frequency (40-60ms) gives smoother streaming. On mobile, increase it (120-200ms) to reduce I/O pressure. Find your sweet spot in the Basic settings tab. The default 80ms is a good starting point for most setups.
🔗
Use Session Tags to Find Chat Notes
After each chat session, #notebrain-session is added to the note's frontmatter. Search for this tag in your vault to quickly find all notes containing AI conversations. Great for reviewing past interactions or continuing previous threads.
🌐
Use a Local Model for Privacy
If you are privacy-sensitive, point the Base URL to a local LLM server (Ollama, LM Studio, or vLLM). All processing stays on your machine. Note that local models may have fewer capabilities than cloud models, especially for tool calling and long context windows.
